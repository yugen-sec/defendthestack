Understanding the OSI Model (With Hacker Examples)

By Rusiru • May 03, 2025

The OSI Model is a fundamental framework for understanding how networks operate — and how attackers can break them. Let’s walk through each of the 7 layers with real-world attack scenarios to make things stick.

Layer 7: Application

This is where users interact with applications like browsers, email clients, etc. Hackers often exploit this layer using phishing emails or web-based malware (e.g., malicious JavaScript).

Layer 6: Presentation

This layer handles encryption, compression, and data translation. Think SSL/TLS attacks — if a cipher is weak or misconfigured, data can be decrypted by attackers using tools like sslsplit.

Layer 5: Session

Attackers may hijack or manipulate sessions between systems. A common example is session hijacking in web apps when cookies are exposed over insecure connections.

Layer 4: Transport

This layer ensures reliable delivery. TCP and UDP live here. Attackers might launch TCP SYN Flood attacks to exhaust server resources or use tools like hping3 to scan for open ports.

Layer 3: Network

IP addresses and routing are managed here. Attackers could spoof IPs or perform ICMP-based reconnaissance like ping sweeps or traceroute to map networks.

Layer 2: Data Link

This layer handles MAC addresses and switches. Common attacks include ARP spoofing (Man-in-the-Middle) or MAC flooding to overflow a switch’s CAM table.

Layer 1: Physical

The most basic layer — the hardware itself. Attackers could perform hardware keyloggers, wiretapping, or simply unplug your Ethernet cable.

Why This Matters

Understanding the OSI model isn’t just a certification checkbox — it helps you think like an attacker and defend more intelligently.

Map tools and techniques to layers, and suddenly packet captures, firewall rules, and sniffed traffic will start to make more sense.
